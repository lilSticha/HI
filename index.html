Git / Системи контролю версій

1. Коротка інформація

Системи контролю версій дозволяють відстежувати зміни в коді та документах, зберігати історію та працювати командою. Git — розподілена система VCS, створена Лінусом Торвальдсом у 2005 році.

2. Основні можливості та переваги Git

  • Відстеження змін та історії комітів
  • Відкотування до попередніх версій
  • Гілки (branches) для паралельної роботи
  • Розподілена структура репозиторіїв
  • Співпраця через віддалені репозиторії (GitHub, GitLab)

3. 10 базових команд Git

  • git init — ініціалізує новий репозиторій
  • git clone <url> — клонування віддаленого репозиторію
  • git status — перевірка стану файлів
  • git add <file> — додавання файлу до staging area
  • git commit -m "повідомлення" — збереження змін з описом
  • git log — перегляд історії комітів
  • git branch — перегляд та створення гілок
  • git checkout <branch> — перемикання на іншу гілку
  • git merge <branch> — злиття гілки з поточною
  • git push — відправка змін у віддалений репозиторій
